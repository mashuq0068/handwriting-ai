# CLAUDE.md

Guidance for working in this repository. Read this first — the repo is **not** a single coherent product, and understanding why will save you confusion.

## ⚠️ The most important thing to know

This codebase currently contains **two overlapping products** that share one React app and one Supabase backend:

1. **Roadmapper** (the original app — "canvas-whisperer") — a collaborative product‑roadmap tool. This is what the **database, hooks, and `/canvas`, `/layout`, `/list`, `/workspace` components are built for.** Items live on a draggable Konva canvas across quarter columns, with workspaces, members, comments, roles, and activity logging.

2. **Quillify** (the newer public face) — a marketing site + product UI for a "turn typed text into realistic handwriting" SaaS. This is what `index.html`, `App.tsx`'s routes, and the `/site` components + most `pages/*` present to users.

These two layers are **only loosely connected**. The Quillify pages are mostly **static / mock UI** (hardcoded sample data, no Supabase wiring yet). The Roadmapper data layer is fully built (schema + RLS + React Query hooks) but its pages are **not currently routed** in `App.tsx`. When you make a change, first decide *which layer* you're touching — they have different design systems, data sources, and maturity.

See [Routing reality](#routing-reality) for exactly what's wired vs. orphaned.

## Tech stack

- **Build:** Vite 5 + `@vitejs/plugin-react-swc`, TypeScript 5.8
- **UI:** React 18, React Router 6, Tailwind CSS 3 + shadcn/ui (Radix primitives in `src/components/ui`)
- **State/data:** TanStack React Query 5, Supabase JS (`@supabase/supabase-js`)
- **Canvas:** Konva + `react-konva` (Roadmapper canvas)
- **Forms/validation:** react-hook-form + Zod
- **Charts:** Recharts (Analytics page)
- **Toasts:** Sonner + shadcn toaster (both mounted)
- **Tests:** Vitest + Testing Library (unit), Playwright (e2e, via Lovable config)
- **Platform:** Generated/managed by **Lovable** (`lovable-tagger`, `@lovable.dev/cloud-auth-js`). Files marked "auto-generated by Lovable" or "automatically generated" should not be hand‑edited.

## Commands

```sh
npm run dev        # Vite dev server on http://localhost:8080
npm run build      # production build
npm run build:dev  # development-mode build
npm run lint       # ESLint (flat config, eslint.config.js)
npm run preview    # preview a production build

Package manager: lockfiles for **npm** (`package-lock.json`) and **bun** (`bun.lockb`) both exist. Pick one and stay consistent; npm is the safe default.

## Environment

`.env` (Vite, so vars must be prefixed `VITE_`):

- `VITE_SUPABASE_URL`
- `VITE_SUPABASE_PUBLISHABLE_KEY`
- `VITE_SUPABASE_PROJECT_ID`

The Supabase client (`src/integrations/supabase/client.ts`) is auto-generated; it reads these and persists the session in `localStorage`.

## Path alias

`@/` → `src/` (configured in `vite.config.ts`, `vitest.config.ts`, `tsconfig`). Import as `@/components/...`, `@/lib/...`, etc.

## Routing reality

`App.tsx` is the only place routes are declared. **Wired routes (reachable today):**

| Path | Page | Layer | Notes |
|------|------|-------|-------|
| `/` | `Landing` | Quillify | Marketing home (hero is actively being designed) |
| `/how-it-works` | `HowItWorks` | Quillify | Static |
| `/templates` | `Templates` | Quillify | Static gallery of handwriting fonts |
| `/pricing` | `Pricing` | Quillify | Static |
| `/login` | `Login` | Quillify/auth | |
| `/register` | `Register` | Quillify/auth | |
| `/dashboard` | `Dashboard` | Quillify | **Mock data** (hardcoded recent docs/hands) |
| `/editor` | `Editor` | Quillify | **Mock/local** handwriting editor (no backend) |
| `/train` | `TrainHandwriting` | Quillify | Static multi-step UI |
| `/assistant` | `AIAssistant` | Quillify | Canned reply, no real LLM call |
| `*` | `NotFound` | — | |

**Pages that exist but are NOT routed (orphaned):**

- `Index.tsx` — the **real Roadmapper app** (Konva canvas + list + inspector, fully wired to Supabase via `useRoadmap`).
- `Analytics.tsx` — Recharts dashboard over roadmap items (uses an "Emerald" color scheme, not the Quillify paper theme).
- `AcceptInvite.tsx` — workspace invitation acceptance flow.
- `Signup.tsx`, `ForgotPassword.tsx`, `ResetPassword.tsx` — additional auth screens.

To bring Roadmapper online you'd route `Index`/`Analytics`/`AcceptInvite` and wrap the tree in `AuthProvider` (see gap below).

## Data layer (Supabase / Postgres)

Schema lives in `supabase/migrations/*` and is mirrored in generated types at `src/integrations/supabase/types.ts`. All access goes through `@/integrations/supabase/client`, wrapped by React Query hooks (`useRoadmap`, `useComments`, `useAnalytics`).

**Tables** (all `public`, all RLS-enabled):

- `profiles` — 1:1 with `auth.users`; auto-created by a `handle_new_user` trigger on signup.
- `workspaces` — top-level container; `owner_id`.
- `workspace_members` / `workspace_invitations` — membership + email-token invites.
- `roadmaps` — belong to a workspace; have a `template_type`.
- `roadmap_items` — the canvas cards: `position_x/y`, `width/height`, `start_date/end_date`, `status`, `priority`, `sort_order`, `owner`, `color`.
- `comments` — threaded on a `roadmap_item`.
- `activity_log` — audit trail per roadmap.
- `user_roles` — global app roles.

**Enums:** `item_status` (`planned | in_progress | completed`), `item_priority` (`low | medium | high`), `app_role` (`admin | member | viewer`).

**Security model:** RLS is enforced via `SECURITY DEFINER` helper functions (`has_role`, `is_workspace_member`, `is_roadmap_member`, `is_roadmap_editor`, `is_comment_member`, `is_comment_editor`, `is_invitation_recipient`, `is_workspace_co_member`). When adding queries, assume the policy requires the user to be a member/editor of the parent workspace/roadmap.

**Bootstrap helpers** (`src/lib/workspace.ts`): `ensureWorkspace`, `ensureRoadmap`, `seedDemoRoadmaps` — called by `useWorkspaceSetup()` so a new user always lands on a populated workspace.

**Timeline math** (`src/lib/timeline.ts`): converts `YYYY-MM-DD` dates ↔ canvas X positions across quarter columns (`parseQuarters`, `dateToCanvasX`). This is the bridge between item dates and Konva coordinates.

## Design system & theming

Tokens are HSL CSS variables in `src/index.css` (`:root`), exposed to Tailwind via `tailwind.config.ts` (`hsl(var(--token))`). Dark mode is class-based (`darkMode: ["class"]`).

There are effectively **two palettes in play** — keep them straight:

- **Quillify "Paper & Ink"** (the active public theme): warm cream `--background`, navy `--primary`, orange `--accent`, blue `--ink-blue`. Plus paper textures and handwriting fonts.
- **Roadmapper "Emerald"**: green-based, referenced directly in `Analytics.tsx` (`hsl(160 84% ...)`) and the `canvas.*` color tokens. This predates the Quillify rebrand.

**Custom utilities** (defined in `index.css`, not Tailwind core):

- Paper backgrounds: `.paper-ruled`, `.paper-grid`, `.paper-dot`
- Handwriting fonts (Google Fonts, imported at top of `index.css`): `.font-hand` (Caveat), `.font-kalam`, `.font-apple`, `.font-shadows`, `.font-indie`, `.font-dancing`, `.font-architect`
- Ink + grain: `.ink-text`, `.grain`
- **Hero handwriting animation:** `.hw-wipe*`, `.hw-flourish`, `.hw-nib`, `.hw-note` (left-to-right "writing" reveal + signed flourish; used by `components/site/HandwrittenNote.tsx`). Honors `prefers-reduced-motion`.
- **Floating pills:** `.float-pill` (uses a `--pill-rot` custom property per pill).

Tailwind animations (`fade-in`, `scale-in`, `slide-up`, accordion) are in `tailwind.config.ts`.

> Note: `components/site/SignatureHero.tsx` is the **older** scribble animation, superseded by `HandwrittenNote.tsx`. It's currently unused.

## Auth

- `contexts/AuthContext.tsx` exposes `useAuth()` → `{ user, session, loading, signOut }`, syncing with Supabase `onAuthStateChange` + `getSession`.
- `components/auth/ProtectedRoute.tsx` gates routes; `AuthLayout.tsx` wraps login/register screens.
- `integrations/lovable/index.ts` provides `lovable.auth.signInWithOAuth("google" | "apple")`, which sets the Supabase session from Lovable-issued tokens.

## Testing

- **Unit:** Vitest (`jsdom`, globals on, setup in `src/test/setup.ts` which polyfills `matchMedia`). Test files: `src/**/*.{test,spec}.{ts,tsx}`. Currently only a placeholder `example.test.ts` exists.
- **E2E:** Playwright via `lovable-agent-playwright-config`. Config is a thin wrapper (`playwright.config.ts`) — add overrides there.

## Conventions

- **Components:** PascalCase files, default-exported function components. shadcn primitives stay in `components/ui` and are generally regenerated, not edited.
- **Styling:** Tailwind utility classes; compose conditionals with `cn()` from `@/lib/utils`. Reach for the CSS-var tokens, not hardcoded colors, so theming stays consistent.
- **Data:** all Supabase access through React Query hooks in `src/hooks`; don't call `supabase` directly from components when a hook exists. Use generated types (`Tables<"...">`, `TablesInsert<...>`) rather than hand-written interfaces.
- **Feedback:** user-facing celebrations/notifications go through `lib/celebrations.ts` / Sonner `toast`, not ad-hoc UI.
- **Imports:** use the `@/` alias.

## Known gaps / things to watch

- **`AuthProvider` is not mounted in `App.tsx`.** Any routed page that calls `useAuth()` (or a hook that does, like `useRoadmap`) will get the default context, not a live session. This is "fine" only because the auth-dependent pages aren't routed yet — wrap the router in `<AuthProvider>` before wiring them up.
- **Quillify product pages are mock UI.** `Editor`, `Dashboard`, `AIAssistant`, `TrainHandwriting` render hardcoded data and have no backend. There is **no handwriting-generation backend** in this repo — the DB is the Roadmapper schema.
- **Dual design systems** (Paper&Ink vs Emerald) coexist; check which palette a screen uses before restyling.
- **Generated files** (`integrations/supabase/*`, `integrations/lovable/*`, most `components/ui/*`) carry "do not edit" headers — change the source/generator instead.
- **Two lockfiles** — avoid committing churn from mixing npm and bun.
